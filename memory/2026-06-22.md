# Long-Term Memory

- En la interpretación actual de descuentos SAP, `unidadCondicion` no se maneja como enum cerrado en el contrato; los valores documentados y observados son `%` para condiciones porcentuales y `PAB` para importes/monto.
- `PAB` significa balboa panameño y en este flujo se usa como unidad monetaria para condiciones por monto, no como porcentaje.
- `ZK94` se interpreta como descuento fijo por unidad/cantidad base, no como porcentaje; el importe negativo enviado desde SAP se normaliza a valor absoluto antes de persistir.
- En el cálculo vigente, el descuento fijo se aplica por unidad y luego el total de línea se multiplica por la cantidad comprada, por lo que si el descuento supera el precio unitario el resultado puede quedar negativo.
- Los rangos del descuento se guardan en `P_DESCUENTO` en `RANGOINI` y `RANGOFIN`; para escalas, el mapper genera una fila por tramo y completa el último rango hasta `999999`.
- Para `ZK94` simple, el payload con `cantidadBase = 1` se persistió con rango `1-999999`; para escalas, la regla esperada es un tramo por cada cambio de cantidad base.
