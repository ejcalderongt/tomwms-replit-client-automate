- En MCP no se debe mezclar contexto con WMS/QAS para validar el reporte de Detalle de Existencias; esa validación pertenece al ecosistema `mposbi`.
- La validación local del reporte de Detalle de Existencias con talla/color quedó bloqueada por la base de datos SQL Server no accesible desde esta máquina, así que no se confirmó en vivo que los ajustes ya se mostraran.
- La ruta local de validación para ese reporte es `/index.php/reportes/detalle_existencias`, con el endpoint de datos en `/index.php/reportes/detalle_existencias/buscar`.
- En el código del reporte ya estaban presentes los campos `ajuste_positivo` y `ajuste_negativo` en backend/vista; lo pendiente al fallar el smoke fue la confirmación contra la DB real, no la existencia de columnas en la UI.
- El 500 del reporte `detalle_existencias/buscar` venía de `Repexistencia_model`: el join a `p_producto_firebase` usaba `f.codigo_sucursal` antes de que `f` estuviera definido, lo que provocaba `SQL Server 4104`.
- El fix fue mover el join de `p_producto_firebase` después de `p_sucursal f`, y hacer que `setFirebaseData()` pueda repoblar `p_producto_firebase` por sucursal cuando el filtro viene vacío.
- Validación posterior: `GET /index.php/reportes/detalle_existencias/buscar` ya responde `200` con JSON en local.
- En esta sesión se confirmó además que `mpos-brain` existe solo como scaffold local en `C:\Users\yejc2\OpenClawWorkspace\mpos-brain` y no contiene archivos todavía; el contexto útil sigue estando en la memoria y en `mcp-brain`.
- En el controller `SapDescuentoController`, `SaveChangesAsync()` persiste todo lo que esté trackeado en el `DbContext`; por eso el flujo de escalas necesita validar duplicados por llave compuesta antes de guardar y traducir `SQL 2601/2627` a un `409` claro.
- En combos SAP, el fallo real puede estar en `P_DESCUENTO_COMBO_DET`: si quedan filas viejas para el mismo `CODDESC`, hay que limpiar por `CODDESC` cuando `reemplazarExistente=true`, aunque la cabecera no exista.
- El patrón de reemplazo quedó extendido en `SapDescuentoController` a simples, escalas y combos: `reemplazarExistente=true` limpia por `CODDESC`, y si no viene el flag el API devuelve `409` cuando un `CODDESC` ya existe y el flujo intentaría un upsert silencioso.
- Regla explicita de Kelvyn: el brain es para contexto, documentacion, aprendizaje e instrucciones reutilizables; no se debe mezclar codigo de aplicacion dentro de `brain/`, `docs/`, `learning/` ni `agents/`.
