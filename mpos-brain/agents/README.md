# agents

Reusable agent instructions and operational notes for mPos / MCP.

## What belongs here

- Durable conventions that should be reused across sessions.
- Validation steps, tracing rules, and replace/update behavior.
- Notes about how a controller or flow should behave.
- References to the application repo and the exact files that were changed.

## What does not belong here

- Application source code.
- Large copied file contents.
- Learning material that should live in the product repo.

## Current rule

- Keep the brain as documentation and context only.
- Keep executable code in the application repository.
- When a behavior change is important, document the rule here and point to the code path that implements it.
