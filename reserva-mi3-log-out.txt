## 4) Por que falla en este escenario

Causa real de negocio/estado de datos:
- La reserva requiere estado 3 (Buen Estado) por configuracion de bodega (idproductoestado=3).
- El stock disponible para esas lineas esta en estado 22 (EN RECEPCION), no en estado 3.
- Resultado: no hay stock aplicable para reservar, aunque haya existencia fisica.

Por que el mensaje confunde:
- El tipificador clasifica por texto y termina etiquetando como SIN_VENCIMIENTO_VALIDO
  porque el mensaje genrico incluye la palabra "vencimiento".
- En estas lineas, vencimiento NO es la causa principal.

## 5) Tenemos fix?

Si, hay fix en 2 niveles.

### 5.1 Fix operativo inmediato (sin despliegue)
- Mover/liberar stock de estado 22 -> estado 3 para los productos del pedido 63605
  siguiendo el proceso funcional autorizado (recepcion/cierre/liberacion), no por SQL directo.
- Reprocesar "Enviar Traslados" y validar Quantity_Reserved_WMS por linea.

### 5.2 Fix de codigo recomendado (diagnostico correcto)
- Introducir motivo explicito cuando existe stock fisico pero en estado distinto al requerido:
  TIPO_NO_RESERVA=ESTADO_PRODUCTO_NO_APLICA
- Ajustar clasificador para no inferir SIN_VENCIMIENTO_VALIDO solo por texto generico.
- Construir mensaje desde evidencia (estado requerido vs estado disponible), no solo string libre.

## 6) Query corta de auditoria reutilizable

-- Resumen para lineas fallidas de un NoEnc
WITH fail_lines AS (
  SELECT d.Line_No,d.Item_No,tp.IdProductoBodega,d.Quantity
  FROM i_nav_ped_traslado_det d
  JOIN trans_pe_enc e ON e.Referencia=d.NoEnc
  JOIN trans_pe_det tp ON tp.IdPedidoEnc=e.IdPedidoEnc AND tp.No_linea=d.Line_No
  WHERE d.NoEnc='63605' AND d.Status=0
), rs AS (
  SELECT IdStock,SUM(Cantidad) AS Reservado FROM stock_res GROUP BY IdStock
)
SELECT f.Line_No,f.Item_No,f.Quantity,
       SUM(CASE WHEN s.IdProductoEstado=3 THEN s.Cantidad ELSE 0 END) AS StockEstado3,
       SUM(CASE WHEN s.IdProductoEstado=3 THEN ISNULL(rs.Reservado,0) ELSE 0 END) AS ResEstado3,
       SUM(CASE WHEN s.IdProductoEstado=3 THEN s.Cantidad-ISNULL(rs.Reservado,0) ELSE 0 END) AS DispEstado3,
       SUM(CASE WHEN s.IdProductoEstado=22 THEN s.Cantidad ELSE 0 END) AS StockEstado22,
       SUM(CASE WHEN s.IdProductoEstado=22 THEN s.Cantidad-ISNULL(rs.Reservado,0) ELSE 0 END) AS DispEstado22
FROM fail_lines f
LEFT JOIN stock s ON s.IdProductoBodega=f.IdProductoBodega
LEFT JOIN rs ON rs.IdStock=s.IdStock
GROUP BY f.Line_No,f.Item_No,f.Quantity
ORDER BY TRY_CONVERT(int,f.Line_No);
