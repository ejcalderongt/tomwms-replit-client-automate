# Release 8.4.2 — La Cumbre — 5 de mayo 2026

> **Nota crítica de propagación**: el release fue **liberado para La Cumbre**
> pero **se mergea en `dev_2028_merge`**, por lo cual TODO cliente que
> reciba esa rama (MAMPA QA, futuros 2028 productivos) hereda los 39 cambios.
> No hay aislamiento por cliente en código (la única segregación es por
> `bodega.*` flags y `i_nav_config_enc`, ver §Capability flags).

> **Fuente**: PDF oficial firmado por Erik Calderón, 89 líneas.
>
> **Mapping a código**: validado contra `data/repos/TOMWMS_BOF` y

> 2026-04-24). Validación de schema sobre `TOMWMS_KILLIOS_PRD_2026` (misma
> rama backend) ejecutada 2026-05-07.

**5 grandes ejes funcionales** del release (no son los 39 items
literales, son los temas):

1. **Reemplazo dañado picking/verificación maduro** — BOF tiene UI propia ahora + HH agrega contraseña de autorización + 4 nuevos params de bodega para restringir/permitir reemplazo. **Esto es el primer movimiento serio del producto sobre BUG-001**.
2. **Parámetros de cambio de ubicación restrictivo** — 3 nuevos flags de bodega + funcionalidad simétrica BOF/HH.
3. **Talla y Color** — feature transversal nuevo. Tabla `producto_talla_color` + FK `IdProductoTallaColor` ya en `stock`, `stock_hist`, `i_nav_transacciones_out`.
4. **Caja Máster en recepción + picking** — soporta tarima padre con cajas hijas detalladas.
5. **Ajustes con borrador + import Excel + centros de costo** — flujo de ajustes de inventario rediseñado.

## Cross-impact con BUG-001

> El release **NO arregla BUG-001**. Pero introduce **5 cambios que tocan
> el mismo módulo**, lo cual obliga a re-evaluar el PLAYBOOK-FIX antes de
> aplicarlo. Detalle:

1. **BOF #1 — Reemplazo desde BOF (NUEVO)**: hasta ahora la marca de "dañado picking" sólo se hacía desde HH. Con esta UI BOF, se duplica el universo de operadores que pueden generar el bug. **Si la nueva pantalla replica la lógica VB.NET defectuosa (`Aplicar_Cambio_Ubic_Estado` con `IdEstadoOrigen=IdEstadoDestino`), el bug se acelera**. Auditar `Reemplazo_Producto_En_Picking` endpoint en `WSHHRN/TOMHHWS.asmx.vb`.
2. **HH #4 — Password de autorización para reemplazo**: contención por fricción. Reduce volumen pero no toca la causa raíz. Si Killios habilita esto, las cifras del BUG-001 (hoy 1,904 líneas/mes pico) bajarán **sin que el bug esté arreglado**, lo cual puede dar falsa sensación de fix.
3. **HH #8 — Restricciones al stock candidato a reemplazo**: filtra QUÉ stock puede usarse como reemplazo (no apto / vencido / lote distinto), pero sigue dejando el dañado original sin descuento.
4. **HH #10 — Cambio ubic + estado + licencia automático**: cambia el patrón de generación de `trans_movimientos`. Las queries forenses del PLAYBOOK-FIX y de `forensic_ola1.js` asumen el patrón legacy. **Re-validar las queries antes de aplicar fix en 2028**.
5. **BOF #11-#15 — Ajustes con borrador + import Excel + agrupar**: NO toca BUG-001 directamente, pero abre una vía operativa para que Killios reconcilie masivamente las 318k UM fantasma vía AJCANTN importado de Excel. Sumar al PLAYBOOK §reconciliación.

**Conclusión cross**: el release pone bases para una segunda iteración del fix BUG-001 (UI BOF para reemplazo + flags bodega para restringir reemplazo + import Excel para reconciliación masiva), pero **el fix lógico de descontar stock al marcar dañado sigue pendiente**. Recomendación: aplicar el PLAYBOOK-FIX ANTES de que el release 8.4.2 se promueva a clientes con BUG-001 activo (Killios, BYB, MERCOPAN).
